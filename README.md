# 📖 插件定位


核心目标：

* 长篇上下文管理
* 自动总结
* 世界书维护
* 长期伏笔管理
* 自动分章节
* AI续写稳定化
* 去AI味润色

适用于：

* 长篇小说
* 多角色剧情
* 世界观复杂作品
* 长线伏笔
* 多章节连续创作

---

# 🧠 插件核心逻辑

插件采用：

```txt id="2zg8j0"
过去 → 现在 → 未来
```

的结构化记忆系统。

---

## 总结（过去）

负责：

```txt id="s1qq0s"
已经发生的剧情
```

---

## 世界书（现在）

负责：

```txt id="l2z9vi"
当前世界状态
```

例如：

* 人物关系
* 当前阵营
* 当前身份
* 世界规则
* 能力体系

---

## 长期伏笔（未来）

负责：

```txt id="m4h14m"
未来剧情规划
```

例如：

* 暗线
* 后续反转
* 主线安排
* 长期剧情目标

---

# ⚙️ 二、首次配置插件

---

# 1. AI生成分支数

位置：

```txt id="r3y6cw"
插件设置 → AI生成分支数
```

作用：

```txt id="x7fh7m"
一次生成几个不同续写版本
```

例如：

```txt id="0nyw7d"
1 = 单版本
3 = 同时生成3个不同续写
```

推荐：

```txt id="vny0q5"
1~2
```

分支越多：

* token消耗越大
* 生成时间越长

---

# 2. 定向续写

位置：

```txt id="i0z9t5"
编辑器底部输入框
```

作用：

```txt id="gcqnyw"
告诉AI这次想怎么续写

一旁有伏笔开关，可将输入的内容植入长期伏笔里，后续概率触发

```

例如：

```txt id="0x5m56"
让男主发现线索
氛围偏悬疑
不要立刻揭晓真相
```

# 📚 四、TXT导入正确流程（非常重要）

很多用户最容易踩坑的地方：

```txt id="3h8x1l"
导入TXT后直接续写
```

这是错误的。

---

# ❌ 错误流程

```txt id="0fj6wr"
导入TXT
→ 直接载入第2章
→ 点续写
```

这样：

```txt id="0kg3cl"
AI实际上看不到前面的章节
```

因为插件发送逻辑是：

```txt id="0tq03q"
总结 + 未总结正文
```

而不是：

```txt id="qvbxdu"
整本小说全文
```

---

# ✅ 正确流程（推荐）

---

## 第一步：导入TXT

进入：

```txt id="h7w5od"
TXT导入 / 章节管理
```

导入小说。

---

## 第二步：自动分章节（可选）

插件支持：

```txt id="7v36wq"
按句尾自动切章
```

建议：

```txt id="g2xwuh"
每章 3000~8000字
```

---

# ⚠️ 第三步

## 先手动点击递归分析全部章节

导入TXT后

```txt id="0l8o6r"
必须先分析全文
```

用于注入人设、背景（类似世界书）

推荐模型：Gemini 3.0 F（GG公益）
注意！注意！注意！注意！此功能会快速消耗额度，一章分析一次，所以推荐一开始只导入小说前几章或是短篇小说，暂时无法一次分析一整本小说。

---

# ⚠️ 第四步（最重要）

## 先手动总结前面的内容



```txt id="0l8o6r"
必须先总结前文
```

推荐：

```txt id="u7vv3u"
选择前几章
→ 点击“小总结”
```

或者：

```txt id="mb6a0o"
按字数总结
```

这样插件会建立：

```txt id="n5v7qj"
剧情记忆
```

AI之后才知道：

```txt id="g0t9wx"
前面发生过什么
```

---

# 📌 为什么必须先总结？

例如：

```txt id="fxc0v8"
正文编辑器只加载了第2章
```

那AI实际上：

```txt id="nknlcf"
只能看见第2章
```

前面章节：

```txt id="4k8v3l"
如果没有总结
AI其实不知道
```

会导致：

* AI失忆
* 剧情断层
* 人设异常
* 上章事件接不上

---

# ✍️ 五、载入章节到正文

位置：

```txt id="3mjlwm"
TXT导入 / 章节管理
→ 载入选定章节到正文
```

作用：

```txt id="u0r2sz"
把章节放进正文编辑器
```

之后：

```txt id="w4x9sm"
AI续写会基于：
总结 + 当前正文
```

进行生成。

---

# 🤖 六、开始续写

进入正文编辑器。

点击：

```txt id="m8sl9q"
AI续写
```

---

# 📌 续写发送逻辑

插件发送给AI的内容：

```txt id="z3yyx4"
【总结】
+
【未总结正文】
+
【世界书】
+
【长期伏笔】
```

这就是插件长篇稳定的核心。

---

# 🌍 七、世界书分析

位置：编辑器右上角

```txt id="s4n5c6"
世界设定 / 人设锁定
```

作用：

```txt id="q7w0u4"
记录当前世界状态
```

例如：

* 当前阵营
* 当前身份
* 当前关系
* 当前修为

⚠️ 注意：

世界书：

```txt id="ry0pjt"
不是剧情总结
```

不要把它当总结库使用。

---

# 🔮 八、长期伏笔

位置：编辑器右上角

```txt id="ecjlwm"
长期伏笔
```

作用：

```txt id="kbr6v4"
记录未来剧情安排
```

例如：

* 后续反转
* 长期暗线
* 主线方向
* 作者规划

---

# ✂️ 九、自动分章节

位置：编辑器右上角“自动总结设置”里

```txt id="tgb6v4"
自动总结设置
→ 自动分章节参数
```

支持：

* 自动切章
* 自动原文归档
* 自动世界书分析

逻辑：

```txt id="suyr7t"
达到目标字数
+
完整句尾
→ 自动成章
```

不会在半句话切断。

---

# 🧠 十、自动总结

位置：编辑器右上角“自动总结设置”里

```txt id="9yuzbt"
自动总结设置
```

建议：

```txt id="tkfczw"
总结触发正文长度：
6000~12000
```

推荐：

```txt id="gz0j0h"
8000~10000
```

---

# 📌 自动总结逻辑

例如：

```txt id="t6o0kt"
正文累计到10000字
```

插件会：

```txt id="v5xjmn"
把前面的剧情压缩成总结
```

之后：

```txt id="g9fpkm"
AI看到：
总结 + 新正文
```

而不是：

```txt id="n6zjlwm"
整本小说
```

这样：

* token更省
* 长篇更稳定
* 不容易失忆

---

# ✨ 十一、自动润色

逻辑：

```txt id="8lnwq6"
AI生成正文
→ 玩家点击保存
→ 自动润色本轮新增正文
```

不会润色：

* 前文
* 总结
* 世界书
* 历史正文

---

# 💾 十二、存档隔离

插件所有内容绑定包括：

* 总结
* 世界书
* 长期伏笔
* 原文章节库
* 自动分析配置

切换故事：

```txt id="1lljlwm"
会自动切换全部内容
```

不会串档。

---

# ⚠️ 十三、推荐工作流（最重要）

推荐：

```txt id="7tkjlwm"
→配置API（正文走总结内置，总结、世界书分析、润色走插件，注意点击自动开关）
→导入TXT
→ 手动世界书分析
→ 手动总结前文
→ 载入当前章节
→ AI续写
→ 自动总结
→ 继续续写
```

---


推荐养成习惯：

```txt id="axjlwm"
定期总结
定期检查世界书
维护长期伏笔
```

这样长篇稳定性会明显提升。
